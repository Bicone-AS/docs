# Documentation project instructions

## About this project

- This is the Bicone documentation site built on [Mintlify](https://mintlify.com)
- Pages are MDX files with YAML frontmatter, written in Norwegian (bokmål)
- Configuration lives in `docs.json`
- Run `mint dev` to preview locally
- Run `mint broken-links` to check links

## Terminology

- Use "område" not "site" when referring to SharePoint sites
- Use "teamområde" not "team site"
- Use "tilganger" not "permissions" or "rettigheter"
- Use "snarvei" not "shortcut"
- Use "Utforsker" for Windows File Explorer, "Finder" for Mac

## Style preferences

- Write in Norwegian (bokmål)
- Use active voice and second person ("du")
- Keep sentences concise — one idea per sentence
- Use sentence case for headings
- Bold for UI elements: Klikk **Innstillinger**
- Code formatting for file names, commands, paths, and code references
- Use kebab-case for file names in Norwegian without special characters (ø → oe, å → aa, æ → ae)

## Content boundaries

- Document SharePoint and Outlook guides for end users
- Document how to connect integrations to Bicone
- Document admin tasks (app approval, etc.)
- Do not document Bicone internal architecture or API
